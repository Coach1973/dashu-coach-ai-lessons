# 跨 AI 工具銜接最佳實踐

## 背景
教練同時使用 Claude、Grok、OpenCode、終端機多個 AI 工具，需要在它們之間無縫交接任務。

## 方法 1：對話記錄 JSONL 檔案交接

每個 Claude Code 對話框都有一個對應的 JSONL 記錄：
```
~/.claude/projects/-Users-bymyway--openclaw/[session-id].jsonl
```

交接時給下一個 AI 這個路徑，讓它讀取再繼續。

## 方法 2：GitHub 經驗庫作為共同知識基底

每次切換 AI 工具時，貼這段萬用提示詞：
```
你好，我是「大樹教練」。請先閱讀我的 GitHub 經驗庫：
https://github.com/Coach1973/dashu-coach-ai-lessons
讀完後告訴我重點，再回答我的問題。
```

## 方法 3：HEARTBEAT.md 作為即時狀態同步

`~/.openclaw/workspace/HEARTBEAT.md` 記錄當前任務進度。
切換對話框前先更新這個檔案，下一個 AI 讀它就能立即同步。

## 各工具分工（教練的實際配置）

| 工具 | 定位 |
|-----|------|
| Claude Code（桌面） | 神經中樞：設計、分析、決策、寫任務書 |
| OpenCode | 程式執行、設定修改、模型測試 |
| Grok（Windows） | 跨平台支援、備用分析 |
| 終端機 | 純 shell 執行，零 AI Token 消耗 |

## 重要提醒
- 對話 context 關閉後一切消失
- 每次達成決定立刻 `git commit`，commit hash 才是唯一可信的記錄憑證
- PROMISES.md 記錄待兌現承諾，開工時第一件事就讀它
