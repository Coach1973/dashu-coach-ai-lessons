# OpenCode 模型沒有回應的診斷流程

## 症狀
選了某個模型，輸入問題後介面一片空白，沒有任何回應。

## 正確診斷方式：先用 curl 測試，不要截圖

### 步驟 1：測試 API 基本連通（不帶 tools）
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST "https://api.example.com/v1/chat/completions" \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"model-name","messages":[{"role":"user","content":"hi"}],"max_tokens":10}'
```
200 = API 正常，問題在別處。

### 步驟 2：測試帶 tools（模擬 OpenCode 真實請求）
```bash
curl -s -X POST "..." \
  -d '{"model":"...","messages":[...],"max_tokens":32000,"stream":true,
       "tools":[{"type":"function","function":{"name":"test","description":"test","parameters":{"type":"object","properties":{}}}}]}'
```

### 關鍵發現：Nemotron Super 49B 的 tool-use bug
帶 tools 時回傳：
```json
{"choices": [{"message": {"content": null, "tool_calls": []}}]}
```
`content: null` → OpenCode 顯示空白。這是模型本身問題，無法從使用者端修復。

**結論：** 需要 function calling 的模型才能在 OpenCode 正常使用。

## 查看 OpenCode 錯誤日誌
```bash
cat ~/Library/Logs/ai.opencode.desktop/opencode-desktop_YYYY-MM-DD_*.log | grep "error\|fail"
```

## 查看 OpenCode 當前 session 內容（不截圖）
```bash
sqlite3 ~/.local/share/opencode/opencode.db \
  "SELECT json_extract(p.data,'$.type'), substr(json_extract(p.data,'$.text'),1,200) \
   FROM part p WHERE p.session_id='ses_xxx' AND json_extract(p.data,'$.type')='text' \
   ORDER BY p.time_created DESC LIMIT 5;"
```
查 session ID：
```bash
sqlite3 ~/.local/share/opencode/opencode.db \
  "SELECT id, title FROM session ORDER BY time_updated DESC LIMIT 5;"
```
