# Token 節約實戰守則

> 來源：2026-04-24~26 實戰血淚總結

## 任務分流表

| 任務類型 | 執行方式 | Token 消耗 |
|---------|---------|-----------|
| shell 指令、改檔案、curl 測試 | Bash 工具直接執行 | 極低 |
| 查看 UI 介面內容 | 讀 SQLite/log 檔案 | 極低 |
| 截圖操作（computer-use） | 最後手段 | 高（每張約 2000 token） |
| 分析、設計、決策 | Claude 對話 | 這才值得花 |

## 具體替代方案

### 不要截圖讀 UI，改用：
- OpenCode session 內容 → `sqlite3 ~/.local/share/opencode/opencode.db`
- 服務狀態 → `launchctl list | grep openclaw`
- API 是否正常 → `curl -o /dev/null -w "%{http_code}" ...`
- Bot 最新發言 → `cat ~/.openclaw/workspace/BOT_MESSAGES.md | tail -20`

### 估算截圖輪數
使用 computer-use 前先問：「這個任務需要幾輪截圖？」
超過 10 輪（約 2 萬 Token）→ 先告知教練再執行。

## 違規案例（2026-04-24）
三機 bot 跨進程互通任務，直接截圖操作 Telegram，燒掉大量 Token 才轉交終端機。
**正確做法：** 寫任務書交給終端機，不消耗 Claude 額度。
