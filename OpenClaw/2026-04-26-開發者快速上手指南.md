# OpenClaw 開發者快速上手指南

> 給接手的開發者或 AI 閱讀，避免重踩已知的坑。

## 系統架構

三機獨立進程，全部運行在同一台 Mac mini 4：

| 機器 | 目錄 | Port | Bot | 服務對象 |
|-----|------|------|-----|---------|
| 1號機（學長） | `~/.openclaw/` | 18789 | @openclaw_macbook4_bot | 大樹教練 |
| 2號機（學弟） | `~/.openclaw-kong/` | 18790 | @CoachWu_openclaw_bot | 孔大哥（峰哥） |
| 3號機（學妹） | `~/.openclaw-peipei/` | 18793 | @coachwu_lenovo_bot | 佩佩老師 |

## 修改身份設定的正確步驟

修改其中一機的身份，必須同步修改以下兩個地方：

1. `[機器目錄]/workspace/SOUL.md` — 人格定義
2. `[機器目錄]/hooks/bot-relay-inbound/handler.js` — 注入身份文字（**優先級高於 SOUL.md**）

只改其中一個 = 沒用。

## 修改後必做：清快取 + 重啟

```bash
# 以 1號機為例
rm -f ~/.openclaw/agents/main/sessions/*.jsonl
echo '{}' > ~/.openclaw/agents/main/sessions.json
launchctl kickstart -k system/com.openclaw.bot.main
```

## 跨機通訊唯一正確方式

寫入共享檔案，讓 cron 每分鐘輪詢：
```
~/.openclaw/workspace/shared-context/BOT_RELAY.json
```
```json
{"messages": [{"from": "學長", "bot": "2", "content": "任務", "status": "pending"}]}
```

`"bot": "2"` = 學弟，`"bot": "3"` = 學妹。**禁止用 sessions_send 跨進程呼叫。**

## 編輯 cron/jobs.json 的注意事項

OpenClaw 會定期寫入 cron/jobs.json（更新 state）。
直接編輯時可能被覆蓋。**正確做法：先停服務，改完再重啟。**

## 常用診斷指令

```bash
# 查服務狀態
launchctl list | grep openclaw

# 查最新 bot 發言
tail -30 ~/.openclaw/workspace/BOT_MESSAGES.md

# 查 relay 任務狀態
cat ~/.openclaw/workspace/shared-context/BOT_RELAY.json
```
