# 2026-04-25~26 三機身份錯亂根本原因深度技術報告

> 作者：Claude Code（克勞德助教）｜補充 Grok 版「Hub-Pattern-三機架構建立.md」的程式碼層根本原因

---

## 症狀
1. 叫「學弟」，「學妹」回應
2. 三機都自稱「學長」
3. 改 SOUL.md 完全無效
4. Kong cron relay 不斷 timeout
5. 1號機誤判 2/3號機在 VPS

---

## 根本原因（程式碼層）

### 原因 1：hook handler.js BOT_ID 標籤對調
```javascript
// 錯誤：kong 的 handler.js
const BOT_ID = '3号機-孔大哥';  // ❌ 應是 2號機

// 錯誤：peipei 的 handler.js  
const BOT_ID = '2号機-佩佩';    // ❌ 應是 3號機
```
影響：BOT_MESSAGES.md 記錄全部標籤錯誤，夥伴感知失真。

---

### 原因 2：bot-relay-inbound 注入錯誤身份（最關鍵）

**這就是為什麼改 SOUL.md 永遠無效的原因。**

`bot-relay-inbound/handler.js` 在每條訊息進來前，把身份文字塞進 context，優先級高於 SOUL.md。

Kong 的 inbound handler 實際注入的是：
```
你是 小龍蝦 3號機（學妹），Bot: @coachwu_lenovo_bot  ← 完全錯誤！
```

**修正後：**
```javascript
'你是 **小龍蝦 2號機（學弟）**，Bot: @CoachWu_openclaw_bot，服務孔大哥（峰哥）。'
```

> ⚠️ 教訓：SOUL.md 改了沒用？第一時間查 `bot-relay-inbound/handler.js`

---

### 原因 3：sessions_send 跨進程路由失敗

三機是獨立進程（ports 18789/18790/18793），sessions_send 只在同一進程內有效。

學妹在大群組有 active session，所以「叫學弟」的訊息被路由到學妹。

**修正：跨機通訊一律用 BOT_RELAY.json**
```json
{"messages": [{"from": "學長", "bot": "2", "content": "任務", "status": "pending"}]}
```

---

### 原因 4：Cron relay 401 → timeout

Kong cron AI 誤以為要自己呼叫 Telegram API，curl 發送沒 Token，一直 401，最後 timeout。

**修正：** cron payload 加入「禁止手動呼叫 Telegram API，直接輸出文字，系統自動發送」

---

### 原因 5：Session 快取不清導致修改不生效

```bash
rm -f ~/.openclaw/agents/main/sessions/*.jsonl
echo '{}' > ~/.openclaw/agents/main/sessions.json
launchctl kickstart -k system/com.openclaw.bot.main
```

## 三條最重要教訓

1. **SOUL.md 改了沒用 → 先查 `bot-relay-inbound/handler.js`**
2. **跨機通訊禁用 sessions_send → 只用 BOT_RELAY.json**
3. **改設定後必須清 session 快取再重啟**
