# 大樹教練 AI 經驗庫

> 記錄所有 AI 相關踩坑、解決方案與最佳實踐。讓每一次失敗都有價值。

---

## 🚀 萬用提示詞（每次開新視窗必貼）

```
你好，我是「大樹教練」。

從現在開始，請你**先做以下動作，再回答我的問題**：

1. 閱讀我的 GitHub 經驗庫：https://github.com/Coach1973/dashu-coach-ai-lessons
2. 優先閱讀：README.md → OpenClaw/ 資料夾 → 通用經驗/ 資料夾
3. 回答前先思考：這個問題我之前踩過類似的坑嗎？有現成正確做法可以套用嗎？

我的目標是「不再重複犯錯」。請確認讀完後告訴我你讀到的重點。
```

---

## 📁 資料夾結構

| 資料夾 | 內容 |
|--------|------|
| `OpenClaw/` | OpenClaw 三機系統相關（身份、架構、hook、relay） |
| `多Agent設計/` | 多 Agent 協作、跨進程通訊、Turn-Taking 設計 |
| `Session與記憶管理/` | Session 快取、記憶讀取、SOUL.md 更新流程 |
| `Prompt-Engineering/` | 提示詞設計、格式、Token 節約 |
| `通用經驗/` | 不分類的通用坑與最佳實踐 |

---

## 📝 新增經驗格式

每份文件包含：
- **問題描述**
- **錯誤做法**（當時怎麼做的）
- **正確做法**（後來怎麼解決的）
- **關鍵教訓**

檔案命名：`YYYY-MM-DD-問題簡述.md`
